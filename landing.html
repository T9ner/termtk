Documentation.

Extensibility Protocol

Terminal v2.0 introduces a deterministic plugin architecture. Modules are written in Rust or TypeScript and compiled to WebAssembly for near-native execution speed within the sandbox.

Unlike traditional terminal emulators, our API exposes the full AST of the command buffer, allowing you to manipulate rendering before the frame hits the GPU.

// terminal.config.ts
import { defineConfig, Plugin } from '@terminal/core';

export default defineConfig({
  performance: {
    gpuAcceleration: true,
    frameRate: 'uncapped'
  },
  plugins: [
    Plugin.load('@official/git-status'),
    Plugin.load('@official/docker-metrics')
  ]
});

Intelligence.

Is the core engine open source?

The plugin API, renderer bridge, and standard library are fully open-source (MIT). The core GPU orchestrator remains proprietary to ensure uncompromised performance and security integrity.

Do you collect telemetry?

Absolutely not. Terminal is an air-gapped instrument by default. We do not track usage, we do not monitor keystrokes, and we have zero network calls outbound unless explicitly invoked by a user-installed plugin.

What operating systems are supported?

Currently calibrated for macOS (Apple Silicon only), Windows 11 (DirectX 12 required), and select Linux distributions running Wayland. We do not support legacy hardware.
