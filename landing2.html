Three ways through.

Most messengers need the internet. Nod works without it. Three connectivity tiers ensure your messages arrive — whatever the conditions.

TIER 01 ◉

Internet Relay

Messages route through a lightweight cloud relay for global reach. Store-and-forward delivery, user search, live presence, and read receipts. Self-hostable.

TIER 02 ◎

Campus LAN

On the same network? Nod discovers peers via UDP broadcast and syncs directly over TCP. No internet required. Zero configuration. It just finds them.

TIER 03 ○

Sneakernet

No network at all? Export your messages to a file. Transfer by USB, email, carrier pigeon — whatever works. Import on the other end. Messages delivered.

Built different.

01.
Offline-First Storage
Every message lives in a local SQLite database. The app works fully without any network. When connectivity returns, sync happens automatically — nothing is ever lost.
02.
Content-Addressed Messages
Every message ID is a SHA-256 hash of its content. Duplicates are impossible. Integrity is guaranteed across all three sync methods — relay, LAN, and sneakernet.
03.
End-to-End Encryption
NaCl box encryption with X25519 key derivation. Your messages are encrypted before they leave your machine. The relay cannot read them. Nobody can.
04.
Single Binary
One CGO-free executable per platform. No runtime dependencies. No installers. No Docker. Download it, run it, and you're chatting. Pure Go, pure simplicity.
05.
Self-Hostable Relay
The relay server is the same single-binary story. Deploy it to Fly.io, your own VPS, or run it on your LAN. Point your clients at it. You own the infrastructure.

Intelligence.

Can the relay read my messages? +

No. Messages are encrypted end-to-end with NaCl box (X25519 + XSalsa20 + Poly1305) before they leave your machine. The relay is a dumb pipe — it stores encrypted blobs and forwards them. It has no access to plaintext.

What happens when I go offline? +

Nothing breaks. Your messages are queued locally in an outbox. When connectivity returns — via internet, LAN, or sneakernet — Nod automatically drains the outbox and syncs everything. Messages are content-addressed, so duplicates are mathematically impossible.

How does LAN discovery work? +

Nod broadcasts UDP packets on port 55557. Any other Nod instance on the same local network will hear the broadcast, exchange peer info, and establish a direct TCP connection for real-time chat and message sync. No internet, no relay, no configuration.

What is sneakernet sync? +

Press Ctrl+E to export your messages as a JSON file. Transfer it physically — USB drive, email attachment, whatever. The recipient presses Ctrl+I to import. SHA-256 content addressing ensures integrity and prevents duplicates.

Is it really a single binary? +

Yes. Nod is written in pure Go with a CGO-free SQLite driver. It compiles to a single executable with zero runtime dependencies — no shared libraries, no Docker, no interpreters. Download it, run it. Works on Windows, macOS, and Linux.

What data does Nod collect? +

None. Zero telemetry, zero analytics, zero network calls except the ones you initiate. Your username, messages, and keys live exclusively in a local SQLite file on your machine. The relay sees only encrypted blobs and IP addresses. There is nothing to harvest.
