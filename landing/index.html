Capabilities.

01.
E2E Encryption.
NaCl box encryption with X25519 key exchange. Messages are encrypted before they leave your device. No server — not even ours — ever sees your plaintext.
02.
Three Connectivity Tiers.
Internet relay for global reach. UDP discovery + TCP sync for campus LAN. Sneakernet JSON export for zero connectivity. Messages always find a path.
03.
Local-First Storage.
SQLite database on your device with content-addressed message IDs (SHA-256). Your data never leaves your machine unless you explicitly send it.
04.
Native Desktop.
Wails v2 with WebView2. A single .exe with no dependencies. Runs on Windows with a native feel — not a browser pretending to be an application.

Intelligence.

Is Nod open source?

Entirely. The full stack — client, relay server, CLI, and desktop app — is MIT-licensed and available on GitHub. We believe communication tools should be auditable.

Do you collect telemetry?

No. Nod is offline-first by design. There are zero network calls unless you explicitly connect to the relay or discover peers on your LAN. No tracking, no analytics, no cloud dependency.

What happens when there's no internet?

Messages queue locally in your outbox and are delivered automatically when connectivity resumes — via relay, LAN sync, or sneakernet file transfer. Nothing is ever lost.

What platforms are supported?

Windows desktop (native .exe via Wails v2), a localhost web UI accessible from any browser, and a terminal TUI built with Bubble Tea. macOS and Linux builds are planned.
